--- layout: default title: Projetos 01 permalink: /projetos/ --- # Projetos Professor: Rodrigo Chaves --- O que a CJR faz? --- Web Applications --- # Web Applications São aplicações em nuvem que tentam solucionar um problema do nosso cliente. Esse problema está relacionado muitas vezes com um processo dentro da empresa que é trabalhoso e precisa ser automatizado. Exemplos: Sistema de Gerenciamento de qualquer coisa... --- Sites --- # Sites Atualmente sites são uma forma básica de divulgação de empresas. Portanto se torna uma necessidade básica para nossos clientes. Nós queremos que nossos clientes tenham esses site que mostre eles para o mundo. --- # Pirâmide de TI Nível Básico: Sites / Emails / Servidor Atividades básicas que todas as empresas tem independentes dos seus serviços. Nível de Automatização: Automação de Processos São ferramentas que automatizam processos dentro da empresa. Isso permite que algumas atividades dentro da empresa sejam realizadas com mais facilmente. Nível Executivo: Tentam tomar a decissão pelos usuários São ferramentas especilizadas que buscam as informações, as analizam e mostram para o usuário qual opção escolher. Usam BI, Machine Learning, Data Mining... --- Quais tecnologias usamos? --- HTML (Slim) CSS (Sass) Javascript (JQuery) Ruby (Ruby on Rails) Postgres entre outras alguns projetos específicos --- # Modelo de Desenvolvimento --- # Equipes 1 Gerente 2 Desenvolvedores (pode aumentar caso seja um projeto mais complicado) --- # Desenvolvimento Ágil + Sprint + Módulo Durante 1 semana, a equipe tem um conjunto de atividades para entregar. Depois de um módulo (geralmente os módulos duram de 1 à 3 sprints, aproximadamente), a equipe tem que entregar uma pequena porção do projeto que entregue alguma funcionalidade útil para o cliente. O cliente válida o que fizemos damos contínuidade ao projeto. --- # Feedback --- # Expectativa --- # Ferramentas --- + Comunicação rápida: Slack ou Whatsapp + Planejamento: Trello ou Taiga + Versionamento: GitHub + Documentação: GoogleDrive --- # Desafios --- + Não ensinamos tudo que vocês precisam saber em um projeto. Só ensinamos os básicos. + Descobrir o que o cliente quer. + Entregar algo que o cliente queria e seja útil e ao mesmo tempo seja possível fazer. --- # Recomendações --- + Usem o stackoverflow. + Não esperem o líderes de vocês.
